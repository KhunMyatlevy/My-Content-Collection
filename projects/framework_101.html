Frame Work 101

အားလုံးပဲ mingalar pr Hacker အေးမောင်ပြန်လာပါပြီ။ ဒီနေ့မှာတော့ Website တွေ Mobile App တွေတည်ဆောက်ရာမှာအသုံးပြုတဲ့ Framework တွေအကြောင်းကိုပြောပြသွားမှာဖြစ်ပါတယ်။ Software တစ်ခုကို develop လုပ်ရာတွင် efficiency and structure ကအရေးကြီးပါတယ်။ Framework ဆိုတာ application တွေတည်ဆောက်ရာတွင်ကျော်ထောက်နောက်ခံအနေနဲ့ပါဝင်ကြပြီး Developer တွေကို maintainable, scalable ဖြစ်တဲ့ app တွေတည်‌ဆောက်နိုင်ဖို့ ready to use tools တွေကိုထောက်ပံပါတယ်။ Developer တွေကဒါတွေကိုသုံးပြီးတော့ application တွေကိုအချိန်ကုန်သက်သက်သာသာနဲ့တည်ဆောက်နိုင်တယ်ပေါ့ပြောရမယ်ဆိုရင်။

What is Framework exactly?
Framework ဆိုတာ pre built tool တွေနဲ့ library တွေကိုစုပေးထားတာဖြစ်ပြီး Developer အနေနဲ့အသုံးပြုပါက application တစ်ခုကိုအချိန်ကုန်သက်သာစွာနိုင်တည်ဆောက်နိုင်စေပါတယ်။ သူကအိမ်တစ်အိမ်ရဲ့ foundation ကိုဆောက်စရာမလိုပဲ ready to use ထားပေးထားတာနဲ့ဆင်တူပါတယ်။ ပြောချင်တာကတော့ Framework ကိုအသုံးပြုလိုက်မယ်ဆိုရင် Developer အနေနဲ့ ကိုယ် application structure ကိုအစကနေတည်ဆောက်စရာမလိုပါဘူး။ အိမ်တစ်အိမ်ကို foundation ချပေးပြီးမှရော့ဆက်ဆောက်လို့ပြောတာနဲ့ဆင်တူပါတယ်။

Types of Frameworks
Frameworks တွေကလဲတစ်ခုခုနှင့်တစ်ခုတူကြပဲကွဲပြားကြပါတယ်။ဒါကလဲတည်ဆောက်နေတဲ့ application ရဲ့ရည်ရွယ်ချက်နှင့်သူကိုအသုံးပြုမယ် platform ပေါ်မှာမူတည်လို့ပါ။
Frontend Frameworks: ဒါကတော့ Front end development မှာအသုံးပြုပါတယ်။ သူက Front end development အတွက်လိုအပ်တဲ့ tool တွေနှင့် library တွေကို ready to use အနေနဲ့ထောက်ပံပေးပါတယ်။ (Example: React, Angular, and Vue.js)
Backend Frameworks: ဒါကတော့ Back end development မှာအသုံးပြုလေ့ရှိကြတယ် Django တို့ Laravel တို့ပဲဖြစ်ပါတယ်။ များတော့အားဖြင့် server-side logic တွေကိုကိုင်တွယ်ရပါတယ်။
Mobile Frameworks: သူကတော့ mobile app တည်ဆောက်ရာတွင် cross-platform ရအောင်အသုံးပြုလေ့ရှိကြပါတယ်။ ပြောချင်တာကတော့ ios ကော့ android ကော့ရ‌အောင်လို့ပေါ့။ (Example: Flutter and React Native)
Testing Frameworks: သူကတော့ကိုယ် application ကို automated testing လုပ်နိုင်စေတဲ့ feature တွေ built-in အနေနဲ့ပါဝင်တဲ့အတွက်အသုံးပြုကြတာများပါတယ်။ (Example: Jest and PHPUnit)

Frameworks တွေကိုဘာလို့သုံးသင့်တာလဲ?
Developer တစ်ယောက်အနေနဲ့ Frameworks တွေကိုအသုံးပြုခြင်းဖြင့်ရရှိနိုင်တဲ့အကျိုးကျေးဇူးတွေကတော့
Faster Development: Application တွေမှာအခြေခံကျတဲ့ routing, authentication, and database management တွေကို pre-built and ready to use အနေနဲ့ထောက်ပံပေးထားတာဖြစ်တဲ့အတွက် development လုပ်ရတာမြန်စေပါတယ်။
Standardized Code: Framework တစ်ခုကိုသုံးမယ်ဆိုရင်ကိုယ်ရဲ့ application ကိုသူရဲ့ structure အပေါ်မှာပဲရေးရတာဖြစ်တဲ့အတွက် code တွေကစနစ်ပိုကျပါတယ်။ ဒါကတစ်ခြားသူတွေနဲ့တွဲလုပ်တာပဲဖြစ်စေကိုယ်ဘာသာလုပ်တာပဲဖြစ်စေ စနစ်ကျတဲ့ project ကနားလည်ရပိုလွယ်ပါတယ်။ ဒီမှာတစ်ခုပြောချင်တာကတော့ developer တစ်ယောက်အနေနဲ့ project ရေးရာတွင် code structure ကိုကိုယ်ရေးနေကြ pattern တစ်ခုထားဖို့အကြံပေးချင်ပါတယ်။

Community Support: နာမည်ကြီးတဲ့ Frameworks အများစုမှာ communities ကြီးကြီးတွေရှိလေ့ရှိတဲ့အတွက် third party libraries တစ်ခြား ready to use third party software တွေပါရှိအတွက်အသုံးပြုကြတာပိုလိုကိုအဆင်ပြေပါတယ်။ နောက်ပြီးတစ်ခုခု error တက်ရင်လဲမေးရမြန်းရတာပိုမိုလွယ်ကူပါတယ်။

Built-in Security: Frameworks အများစုမှာအရေးကြီးတဲ့ data တွေကိုကာကွယ်ပေးဖို့ Security feature တွေပါလေ့ရှိပါတယ်။

Challenges When Using Frameworks
Frameworks တွေက developer တွေကိုအကျိုးရှိစေပေမယ်တစ်ချိန်ထဲမှာပဲ cons အနည်းငယ်ရှိပါသေးတယ်ဒါတွေကတော့
Learning Curve: Frameworks တွေကလဲ Language တစ်ခုလောက်တော့လေ့လာရတာမခက်ပေမယ်အတိုင်းတာတစ်ခုတော့ခက်ပါတယ်။ အထူးသဖြင့် Frameworks အသစ်တစ်ခုကိုခုမှစလေ့လာတဲ့လူအတွက်ဆိုရင်ပေါ့။
Dependency Risks: Frameworks တစ်ခုကိုပဲအသုံးပြုနေမယ်ဆိုရင်ကြာလာတဲ့အခါတစ်ခြား Frameworks တစ်ခုကိုပြောင်းတဲ့အခါ အခက်ခဲရှိနိုင်ပါတယ်။ ဒါကတော့ Frameworks မှမဟုတ်ဘယ်ဟာမဆိုတစ်ခုကိုအားကိုးပြီးအသုံးပြုတာအချိန်ကြာရင်ဖြစ်တတ်လေ့ရှိပါတယ်။
Overhead: Project ကသေးနေတဲ့အခါမှာတစ်ခါတစ်လေ Frameworks သုံးတာကလဲ overkill ဖြစ်စေနိုင်ပါတယ်။ အထူးတစ်ဖြစ် large application တွေကိုတည်ဆောက်ဖို့သုံးတဲ့ Frameworks တွေကို application သေးသေးလေးတွေမှာသုံးတာကလိုတာထက်အလုပ်ပိုရှုပ်ပါတယ်။
