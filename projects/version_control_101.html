Version Control 101

Understanding Version Control: A Guide to Managing Code Effectively

What’s up! Tech Nerds! Hacker အေးမောင် ပြန်လာပါပြီ။ ဒီနေ့မှာတော့ လူတွေသိပ်မပြောကြပေမယ့် programmer တွေအတွက်အရေးကြီးတဲ့ version control အကြောင်းပြောပြသွားမှာဖြစ်ပါတယ်။

What is version control?
Version control ဆိုတာကိုယ် project ထဲက file တွေကို track ပေးတဲ့ system လို့အလွယ်မှတ်လို့ရပါတယ်။ File တွေကို track နိုင်ခြင်းဖြင့် file ထဲက code အပြောင်းလဲမှန်သမျှကို record ယူနိုင်သည်အပြင် code မှားဖျက်မိတာ overwrite မိတာတွေကိုပြန်ပြင်နိုင်ပါသေးတယ်။ ဒါအပြင် branch တွေသုံးပြီးမတူညီတဲ့ developer ၂ယောက်က code file တစ်ခုထဲကိုတစ်ပြိုင်တည်းရေးနိုင်ပါတယ်။ပြီးမှာပိုကောင်းတဲ့လူရဲ့ code ကို merge လိုက်ယုံပါပဲ။ အသေးစိတ်ကိုအောက်မှာဆက်ရှင်းပြပါမယ်။

The problems without version control
Version control ကမဖြစ်မနေသုံးရမယ်မဟုတ်ပေမယ့်သူကိုမသုံးရင် developer တစ်ယောက်အနေနဲ့အလုပ်လုပ်ရတာခက်ပါတယ်။ အထူးသဖြင့် large scale application တွေကိုကိုင်တွယ်ရပြီဆို version control မသုံးထားရင်တော်တော်တိုင်ပတ်ပါတယ်။ Code တွေမှားဖျက်တာ၊ မှားပြီးပြန်ရေးမိတာတွေဖြစ်ရင် back up code မရှိတဲ့ခါမှအကုန်ပြန်ရေးရပါတယ်။

Key concepts of version control

Repository
Repository ဆိုတာကိုယ် project ရဲ့ code တွေအပြင် code history ကိုပါသိမ်းပေးထားတဲ့ file or folder လို့မှတ်လို့ရပါတယ်။ Repository မှာ Local repository နဲ့ Remote repository ဆိုပြီး ၂မျိုးရှိပါတယ်။ Local repository ဆိုတာ local ဆိုတဲ့အတိုင်းကိုယ်ရဲ့ PC or machine မှာပဲသိမ်းထားတာဖြစ်ပြီး Remote repository ကတော့ GitHub တို့လို platform တွေမှာသိမ်းတာကိုပြောတာပါ။ Local repository တစ်ခုဖန်တီးဖို့ Git ကိုသုံးရပါတယ်။ Remote repository ကိုတော့ Git ကိုသုံးပြီးဖန်တီးလို့ရသလို GitHub ကိုသုံးပြီး drag and drop နဲ့လဲ file တွေကိုသိမ်းလို့ရပါတယ်။

Commit
Commit ကိုအလွယ်ပြောရရင်တော့ code အပြောင်းလဲတွေကို save ထားပေးတဲ့ screenshot or snapshot လို့သတ်မှတ်လို့ရပါတယ်။ Code တစ်ခုကို overwrite တာဖြစ်ဖြစ်၊ အသစ်တွေထည့်တာဖြစ်ဖြစ် commit လုပ်နိုင်ပြီး commit မှာ comment တစ်ခုထည့်နိုင်ပါတယ်။ Commit လုပ်လိုက်ရင် code က repository ထဲမှာ ပြောင်းလဲသွားပြီး အဲဒီအပြောင်းလဲတွေကို မှတ်တမ်းတင်ထားမှာဖြစ်ပါတယ်။

Branch
Branch ဆိုတာကိုနားလည်လွယ်အောင်ဥပမာပေးရမယ်ဆိုရင် Project မှာ feature အသစ်ထည့်ချင်တယ်ဆိုပါစို့။ ရှိပြီးသား code တွေကိုမှားဖျက်မိတာမှရှောင်ချင်ရင် Branch တစ်ခုဖန်တီးလိုက်ယုံနဲ့ feature အသစ်ကိုရေးနိုင်ပါတယ်။ Feature အသစ်ကအဆင်ပြေတော့မှ မူရင်း project နဲ့ merge လုပ်ပါတယ်။

Merge
Merge ကတော့ Branch မှာရေးထားတဲ့ code တွေကို မူရင်း project မှာပေါင်းထည့်ဖို့သုံးရတာပါ။ Branch တစ်ခုနှင့်တစ်ခုကိုပေါင်းချင်ရင်လည်း merge ကိုသုံးပါတယ်။

Pull and Push
Pull ဆိုတာ Remote repository က file တွေကို Local repository ထဲကိုထည့်ချင်ရင်သုံးပါတယ်။ Push ဆိုတာ Local repository က file တွေကို Remote repository ထဲကိုထည့်ချင်ရင်သုံးပါတယ်။

Summary
Version Control System ဆိုတာ အသုံးပြုရတာလွယ်ကူပြီး developer တွေအတွက်အသုံးဝင်တဲ့ tool တစ်ခုဖြစ်ပါတယ်။ Project အကြီးကြီးတွေလုပ်တာဖြစ်ဖြစ်၊ အချင်းချင်း collaborate လုပ်တဲ့ project တွေမှာမဖြစ်မနေသုံးသင့်တဲ့ tool ပါ။ ကျနော်ကတော့ project ဘယ်သေးသေးကြီးကြီးမဆို version control ကိုသုံးဖို့အကြံပေးလိုက်ပါတယ်။ ဒီ page မှာ programming နဲ့ပတ်သက်တဲ့ content တွေတင်ပေးသွားမှာဖြစ်တဲ့အတွက် like and follow လုပ်သွားပါဦး။ Post ကိုလည်း ကြိုက်နှစ်သက်ရင် share လုပ်ဖို့မမေ့ပါနဲ့နော်။ ဒီနေ့တော့ဒီလောက်ပါပဲ။
